AI Workflow Pipeline

How InsightFlow thinks.

Every file you drop goes through a structured 7-stage pipeline. Here is exactly what happens between upload and insight.

Stage 01
File Upload
User drops .xlsx, .csv, .json, .tsv, or .txt. File is read entirely in the browser — nothing is transmitted.
Stage 02
Parse & Clean
RFC 4180-compliant CSV tokenizer handles quoted fields, commas in text, and special characters. Rows are validated and empty entries filtered.
Stage 03
Classify Columns
Each column is tested for numeric parseability. Columns with ≥60% numeric values become metrics. The rest become category labels for chart axes.
Stage 04
Compute Statistics
Mean, median, standard deviation, min, max, Q1, Q3, growth rate, and anomaly count computed per numeric column. Anomalies flagged at >2σ from mean.
Stage 05
Detect Trends
First and second halves of each series are compared. If the second half average exceeds the first by >10%, the trend is marked upward. Below −10% is downward.
Stage 06
Generate Charts
Charts are built dynamically from detected columns — not templates. Numeric columns become Y-axis values; categorical columns become labels. Chart type is user-switchable.
Stage 07
Surface Insights
A plain-English narrative is generated from the computed statistics. Language is shaped by the selected business context (Sales, Finance, SaaS, etc.) to keep insights domain-relevant.
Upload ──────→ Parse ──→ Classify ──→ Statistics ──→ Trends ──→ Charts ──→ Insights ✓
